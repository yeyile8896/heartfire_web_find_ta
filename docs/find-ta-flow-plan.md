# 《爱，怎么翻译》营会互动平台｜寻找那个 TA 流程规划

版本：v0.1  
日期：2026-06-18  
状态：流程规划初稿，后续用于逐项扣细节

## 1. 活动定位

「寻找那个 TA」是营会前期破冰活动。核心目标不是让大家先认识名字，而是先通过匿名线索、现场观察和双人任务，建立第一份真实连接。

活动不要求下载 App，营员只需要用手机浏览器扫码进入房间。主持人使用后台控制配对和查看进度，大屏显示二维码、匿名代号墙、积分榜和现场动态。

## 2. 参与角色

### 营员

营员通过二维码进入房间，填写自己的真实信息和匿名线索。

营员可见：

- 自己的匿名代号
- 神秘伙伴的匿名代号
- 神秘伙伴提供的线索
- 自己组合的积分和任务状态

营员不可见：

- 对方真实姓名
- 对方小组
- 后台配对表
- 其他组合的真实信息

### 主持人 / 管理员

主持人负责创建房间、展示二维码、开始随机配对、查看真实配对表、观察任务进度和排行榜。

主持人后台可见：

- 真实姓名
- 小组
- 匿名代号
- 配对结果
- 会合确认状态
- 任务完成状态
- 当前积分

### 主持人大屏

大屏适合投影到现场，用来营造类似 Kahoot 的集体参与感。

大屏显示：

- 房间码
- 加入二维码
- 已加入人数
- 已配对人数
- 已完成任务人数
- 匿名代号墙
- 排行榜 Top 10
- 实时滚动动态

大屏不显示：

- 真实姓名
- 小组
- 后台配对关系
- 私密身份信息

## 3. 总体活动流程

### 阶段 0：主持人准备

主持人进入「寻找那个 TA」后台，创建一个活动房间。

系统生成：

- 房间码
- 营员加入二维码
- 主持人大屏链接
- 主持人后台链接

主持人将大屏投影到现场，营员开始扫码加入。

### 阶段 1：营员扫码加入

营员打开手机浏览器扫码，进入加入页面。

营员填写：

- 真实姓名
- 小组
- 匿名代号
- 兴趣爱好
- 今日穿搭
- 确认暗号
- 给 TA 的一句话

提交后，营员进入自己的个人任务页。

加入后，大屏显示该营员的匿名代号，例如：

- 匿名 01
- 匿名 02
- 匿名 03

### 阶段 2：主持人开始随机配对

当人数差不多到齐后，主持人点击「开始随机配对」。

系统将在线营员随机分成 2 人一组。

默认规则：

- 2 人一组
- 如果出现单数，可以后续决定是生成 3 人组，还是让最后一人等待下一轮
- 配对结果只在后台显示真实姓名
- 营员页面只显示神秘伙伴的匿名线索

### 阶段 3：匿名寻找

配对后，营员个人页面显示：

- 自己的匿名代号
- 神秘伙伴的匿名代号
- 神秘伙伴的兴趣爱好
- 神秘伙伴的今日穿搭
- 神秘伙伴的确认暗号
- 神秘伙伴的一句话

营员需要在现场根据这些线索找到 TA。

允许分享：

- 兴趣爱好
- 穿搭细节
- 指定暗号
- 模糊经历

禁止分享：

- 真实姓名
- 照片
- 房间号
- 手机号
- 社交媒体账号
- 其他明显身份信息

### 阶段 4：双方确认会合

两个人现实中找到彼此后，需要完成：

- 互相确认暗号
- 击掌
- 各自在页面点击「我已找到 TA」

确认规则：

- 第一人点击后，页面显示：等待 TA 确认
- 另一人点击后，系统才判定会合成功
- 会合成功后获得 10 分
- 积分从 0 / 80 变为 10 / 80
- 大屏排行榜同步更新
- 后续双人任务自动解锁

### 阶段 5：任务 1｜共同点挑战

解锁条件：

- 双方都已确认会合

任务内容：

- 填写 1 个共同点
- 填写 1 个不同点
- 「最意外的发现」为可选项

完成方式：

1. 任意一方填写共同点挑战表单。
2. 提交后，提交者自动确认。
3. 另一方页面显示同一份答案。
4. 另一方点击「确认这份答案」。
5. 双方确认后，任务完成。
6. 组合获得 20 分。

积分变化：

- 完成会合后：10 / 80
- 完成共同点挑战后：30 / 80

### 阶段 6：任务 2｜默契测试

计划功能，后续细化。

建议玩法：

- 系统随机生成 10 道二选一题
- 双方各自在手机上作答
- 作答期间不显示对方答案
- 双方完成后，系统计算默契度

题目示例：

- 咖啡 / 奶茶
- 海边 / 山上
- 早起 / 晚睡
- 电影 / 运动
- iPhone / Android
- 安静 / 热闹
- 甜 / 咸
- 计划型 / 随性型

完成规则建议：

- 双方都提交答案后自动完成
- 系统计算默契度百分比
- 完成后获得 20 分
- 默契度可进入大屏特别榜单

积分变化：

- 完成默契测试后：50 / 80

### 阶段 7：任务 3｜营地探索

计划功能，后续细化。

建议玩法：

系统为每组随机生成一个探索任务。

任务示例：

- 找工作人员合照
- 找另一组搭档合照
- 找指定地点拍照
- 找到一个营会标志物
- 完成一个现场小采访

完成方式建议：

- 两人一起完成任务
- 上传照片或由工作人员扫码确认
- 主持人后台可审核是否通过

完成后获得 30 分。

积分变化：

- 完成探索任务后：80 / 80

### 阶段 8：排行榜与收尾

排行榜实时展示：

- 当前积分 Top 10
- 已完成任务数
- 默契度最高组合
- 最快会合组合

收尾时主持人可以宣布：

- 第一名组合
- 默契度最高组合
- 最快找到 TA 组合
- 最有趣共同点组合

## 5. 页面规划

### 营员加入页

路径示例：

`/find-ta/join/房间码`

功能：

- 填写真实姓名
- 填写小组
- 填写匿名代号
- 填写匿名线索
- 提交后进入个人任务页

### 营员个人页

路径示例：

`/find-ta/room/房间码/me/营员ID`

功能：

- 查看自己的匿名代号
- 查看神秘伙伴线索
- 点击「我已找到 TA」
- 查看组合积分
- 完成共同点挑战
- 后续完成默契测试
- 后续完成营地探索

### 主持人后台

路径示例：

`/find-ta/host?room=房间码&token=主持人token`

功能：

- 创建房间
- 显示二维码
- 开始随机配对
- 查看真实配对表
- 查看每组确认状态
- 查看每组任务状态
- 后续导出数据

### 主持人大屏

路径示例：

`/find-ta/screen/房间码`

功能：

- 显示二维码
- 显示房间码
- 显示匿名代号墙
- 显示实时数据
- 显示排行榜
- 显示现场动态

## 6. 状态设计

### 营员状态

| 状态 | 说明 |
| --- | --- |
| 等待中 | 还没有开始配对 |
| 已配对 | 已经拿到神秘伙伴线索 |
| 等 TA 确认 | 自己已确认会合，对方未确认 |
| TA 已确认 | 对方已确认会合，自己未确认 |
| 已会合 | 双方都确认成功 |

### 共同点挑战状态

| 状态 | 说明 |
| --- | --- |
| 会合后解锁 | 还没有双方确认会合 |
| 可填写 | 已会合，可以提交共同点挑战 |
| 等待 TA 确认 | 自己提交或确认后，等待对方确认 |
| 请确认 | 对方已提交，自己需要确认 |
| 已完成 | 双方确认完成并加分 |

## 9. 需要继续确认的细节

### 配对规则

需要确认：

- 是否允许同小组配对
- 是否希望尽量跨小组配对
- 是否需要男生女生分开或混合
- 单数人数时，是生成三人组还是等待下一轮
- 迟到营员是否可以加入下一轮

### 匿名聊天

需要确认：

- 是否真的需要网页内聊天
- 还是只靠线索和现场找人
- 是否需要禁词检测真实姓名、手机号、社交账号
- 主持人是否需要查看聊天记录

### 共同点挑战

当前规则：

- 必填 1 个共同点
- 必填 1 个不同点
- 最意外的发现可选
- 双方确认后加 20 分

后续可确认：

- 是否允许提交后修改
- 是否需要主持人审核
- 是否要在大屏展示精选共同点

### 默契测试

需要确认：

- 题目数量
- 题库内容
- 是否随机题目
- 是否显示默契度百分比
- 是否进入排行榜特别奖

### 营地探索

需要确认：

- 是否上传照片
- 是否需要工作人员审核
- 任务是随机发放还是所有人一样
- 是否需要限制上传次数

### 数据保存

需要确认：

- 活动当天是否必须断网可用
- 是否使用本地数据库
- 是否需要公网域名访问
- 是否需要活动结束后导出 Excel
